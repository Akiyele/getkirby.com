## New `access` and `list` page permissions

### Changes

- `read` option will deprecated in v5
- `access` option is about accessibility
- `list` option is about visibility (picker and sections)
- `access` and `list` permissions depend on the read option until v5

### Meaning

```yaml
access: false # not accessible, not listed
list: false   # accessible, not listed
read: false   # not accessible, not listed
```

### Sample Usage

```yaml
# Orders page is no accessible and no visible for all roles except admins.

title: Orders

options:
  access:
    *: false
    admin: true
  list:
    *: false
    admin: true
```
