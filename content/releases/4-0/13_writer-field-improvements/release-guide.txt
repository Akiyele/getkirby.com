## Better toolbar

New `toolbar` option [#80](https://github.com/getkirby/v4/issues/80)

```yaml
text:
  type: writer
  marks:
    - bold
    - italic
    - link
    - code
  toolbar:
    inline: false
    marks:
      - bold
      - "|"
      - link

```

`inline` control the position/style of the toolbar.

(image: toolbar.png)

`marks` and `nodes` control what is shown in the toolbar, while the top-level options of the same name define if the marks/nodes are actually supported. So you could, for example, support the code mark for pasting, but hide it from the toolbar.

## Set heading levels

You can set the available heading levels in the writer field with the new `headings` option [#18](https://github.com/getkirby/v4/pull/18)

```yaml
fields:
  text:
    type: writer
    headings:
      - 2
      - 3
      - 4
```

## Counter

Added new `minlength`, `maxlength` and counter options, same behavior as for the `textarea` field. [#83](https://github.com/getkirby/v4/pull/83)

## `sub` and `sup` marks

[**#88**](https://github.com/getkirby/v4/pull/88)

```yaml
fields:
  text:
    type: writer
    marks:
      - sub
      - sup
```

### Clear formats

- New writer field mark that clears all formatting marks (of the default ones all but email and link). 
- New `clear` icon

### Custom marks + nodes extension

- New `writerNodes` Panel extension [#79](https://github.com/getkirby/v4/pull/79)
- New `writerMarks` Panel extension [#79](https://github.com/getkirby/v4/pull/79)

The defined object extends [https://github.com/getkirby/v4/blob/v4-develop/panel/src/components/Forms/Writer/Mark.js](https://github.com/getkirby/v4/blob/v4-develop/panel/src/components/Forms/Writer/Mark.js) [https://github.com/getkirby/v4/blob/v4-develop/panel/src/components/Forms/Writer/Node.js](https://github.com/getkirby/v4/blob/v4-develop/panel/src/components/Forms/Writer/Mark.js)  and thus can make use of the same extension layer our default marks use.
